Como o po-upload monta o payload internamente

O componente po-upload sempre envia os arquivos via HTTP POST usando FormData. O serviço PoUploadBaseService.upload() é responsável pela montagem.

Para cada arquivo selecionado, uma requisição HTTP separada é feita (não é batch).

1. Arquivo único (padrão)

Propriedades: apenas p-url

FormData:
  files: (binary) arquivo.pdf

2. Múltiplos arquivos

Propriedades: p-url + p-multiple

Cada arquivo gera uma requisição POST separada com seu próprio FormData.

# Requisição 1:
FormData:
  files: (binary) arquivo1.pdf

# Requisição 2:
FormData:
  files: (binary) arquivo2.png

3. Com additionalData (event.data)

Propriedades: p-url + (p-upload) event handler

No evento (p-upload), definir event.data = {{'{'}}...{{'}'}}. O objeto é serializado como JSON e adicionado no campo "data". 

FormData:
  files: (binary) arquivo.pdf
  data: "{{'\"userId\":\"123\",\"category\":\"documents\"'}}"

4. Com extraFormData (event.extraFormData)

Propriedades: p-url + (p-upload) event handler

No evento (p-upload), definir event.extraFormData = {{'{'}}...{{'}'}}. Cada chave do objeto vira um campo separado no FormData. 

FormData:
  files: (binary) arquivo.pdf
  token: "abc-xyz-456"
  projectId: "P001"
  data: "{{'{}'}}"

5. Upload de diretório

Propriedades: p-url + p-directory

Cada arquivo do diretório é enviado em requisição separada. O atributo webkitdirectory é adicionado ao input.

6. Headers customizados

Propriedades: p-url + p-headers

Os headers são passados para HttpHeaders e incluídos na requisição POST.

Headers:
  Authorization: Bearer token123
  X-Custom-Header: valor-custom

FormData:
  files: (binary) arquivo.pdf

7. formField customizado

Propriedades: p-url + p-form-field="documento"

O nome do campo no FormData muda de "files" para o valor definido.

FormData:
  documento: (binary) arquivo.pdf

8. Auto-upload

Propriedades: p-url + p-auto-upload

O payload é o mesmo dos cenários acima, mas o envio ocorre automaticamente ao selecionar o arquivo.

9. Com restrições

Propriedades: p-url + p-restrictions

As restrições (extensão, tamanho, quantidade) são validadas ANTES do envio. Arquivos que não atendem às restrições não geram payload.

10. Combinação completa

Todas as propriedades combinadas. Cada arquivo gera um FormData contendo: arquivo + data (JSON) + campos do extraFormData + headers customizados + formField customizado. 

POST /upload-intercepted
Headers:
  Authorization: Bearer token123

FormData:
  attachments: (binary) arquivo.pdf
  token: "abc-xyz-456"
  data: "{{'\"userId\":\"123\"'}}"
