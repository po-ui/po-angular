Cenarios de Token --background
Valida que o token --background cascateia para header E body em todos os page types.
B1 - Token --background em po-page-default
Esperado: header e body com background #e8f5e9 (verde claro).
Conteudo do body - deve ter o mesmo background do header.
Se o header e o body estao com fundo verde claro, o token esta funcionando.
B2 - Token --background em po-page-default (secondary)
Esperado: header secondary e body com background #e3f2fd (azul claro).
Conteudo do body - deve ter o mesmo background do header.
Se o header e o body estao com fundo azul claro, o token esta funcionando.
B3 - Token --background em po-page-default (tertiary)
Esperado: header tertiary e body com background #fff3e0 (laranja claro).
Conteudo do body - deve ter o mesmo background do header.
Se o header e o body estao com fundo laranja claro, o token esta funcionando.
B4 - Token --background em po-page-list
Esperado: header e body com background #fce4ec (rosa claro).
Conteudo da lista - deve ter o mesmo background do header.
Se o header e o body estao com fundo rosa claro, o token esta funcionando.
B5 - Token --background em po-page-detail
Esperado: header e body com background #f3e5f5 (roxo claro).
Conteudo do detalhe - deve ter o mesmo background do header.
Se o header e o body estao com fundo roxo claro, o token esta funcionando.
B6 - Token --background em po-page-edit
Esperado: header e body com background #e0f7fa (ciano claro).
Conteudo da edicao - deve ter o mesmo background do header.
Se o header e o body estao com fundo ciano claro, o token esta funcionando.
B7 - Sem token --background (fallback)
Esperado: usa fallback --color-page-background-color-page (cor padrao do tema).
Conteudo sem token customizado.
Deve usar a cor padrao do tema ativo (light, dark ou sunset).
