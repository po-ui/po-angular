Cenarios de Validacao de Layout Flexbox
Valida que a migracao para Flexbox mantem a integridade visual em diferentes estruturas.
F1 - po-page-default standalone (sem .po-wrapper)
Esperado: min-height 100vh, scroll no content area.
Conteudo dentro do po-page-content.
Deve ocupar espaco restante apos o header.
F2 - po-page-default com conteudo longo (scroll)
Esperado: header fixo no topo, scroll apenas no content.
@for (item of longContent; track item) {
{{ item }}
}
F3 - Header secondary com conteudo longo
Esperado: header secondary fixo, scroll no content.
@for (item of longContent; track item) {
{{ item }}
}
F4 - Header tertiary com conteudo longo
Esperado: header tertiary fixo, scroll no content.
@for (item of longContent; track item) {
{{ item }}
}
F5 - po-page-list (componente consumidor)
Esperado: layout flexbox herdado, header e content corretos.
Conteudo da lista.
F6 - po-page-detail (componente consumidor)
Esperado: layout flexbox herdado, acoes back/edit/remove.
Conteudo do detalhe.
F7 - po-page-edit (componente consumidor)
Esperado: layout flexbox herdado, acoes save/cancel.
Conteudo do formulario de edicao.
F8 - Overflow de acoes nos 3 tipos de header
Esperado: dropdown renderizado sem quebrar layout flexbox.
Primary com 5 acoes (overflow no dropdown).
Secondary com 5 acoes (overflow no dropdown).
Tertiary com 5 acoes (overflow no dropdown).
